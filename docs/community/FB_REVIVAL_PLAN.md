## STAGE 0 — Pre-content work

### Step 1 — Winning-Trades Post Review (HARD GATE)

**Why this gate exists.** One or more historical posts use "winning trades" language.
Under FTC Endorsement Guides and the enforcement record against trading educators
(RISK_REGISTER §17), any post that states, implies, or suggests a profitable outcome —
even without a dollar amount, even in passing — is a potential earnings claim. The FTC
does not require intent; the consumer's reasonable impression controls. This gate must
be cleared before the page goes active in any form.

**Decision rule for each post.**

First: does this post state, imply, or suggest a specific profitable outcome, win rate,
or trading success? Triggers include:

- Any explicit profit figure or percentage gain
- Language like "won," "called it," "nailed it," "winning trade," "green," "profit,"
  "I made," "up X," "correct again"
- A screenshot or image that shows a profit balance, green P&L, or account growth
- Any framing that implies a reader could replicate the result
- Before-and-after account value comparisons
- Success stories attributed to TradeGame coaching or content

If the post triggers any of the above: the default action is DELETE.

**When a disclaimer suffices instead of deletion.**

Deletion is the default because it eliminates the exposure entirely. Editing and adding
a disclaimer is appropriate only when all three of the following are true:

1. The post contains educational content of genuine standalone value (a concept, a
   process explanation, a resource) that would be lost if deleted.
2. The earnings-claim element is separable — it can be removed by editing the post body
   without gutting the educational content.
3. After editing, the post no longer triggers any of the "state, imply, or suggest"
   criteria above.

If a disclaimer is used (because all three conditions are met), add this block verbatim
at the top of the post body:

> Note: This post was edited to remove language that could be read as implying a
> trading outcome. TradeGame is education only — no signals, no financial advice, no
> performance promises. Past discussion here does not reflect typical results.
> Trading involves risk of significant loss.

Do not use a disclaimer as a shortcut to avoid deletion. When in doubt: delete.

### Step 2 — Rename Check (no prerequisite)

This step can be done in the same session as Step 1 or separately. It does not block
Step 1 and Step 1 does not block it — they are independent.

**Goal.** Confirm whether the page display name and username (URL handle) can be set
to TradeGame branding. Target: display name "TradeGame," username "tradegame" or
"tradegameofficial" (check availability).

**Steps (Chromebook browser or any browser).**

1. Go to the Facebook page. Sign in with the admin account.
2. Click "Settings" in the left-side admin menu (may appear as a gear icon or
   "Settings & Privacy").
3. In General settings, find "Name" — the public display name of the page.
   - If editable: type "TradeGame" and save. Note: Facebook may require a review period
     for name changes and limits how often you can change a page name.
   - If greyed out or locked: note the reason shown and record it in the owner-intake
     table in SOCIAL_AUDIT_NOTES.md.
4. Find "Username" — this sets your page URL (facebook.com/[username]).
   - If editable: type "tradegame" and check availability. If taken, try
     "tradegameofficial" or "tradegame.game". Record what you set.
   - If greyed out or locked: note the reason.
5. Record the outcome in the owner-intake table in SOCIAL_AUDIT_NOTES.md:
   - Current display name (whatever it shows before any change)
   - Whether rename was possible
   - What name and username were set (or what the lock reason was)

NEEDS-HUMAN: browser access with page admin login.

### Step 5 — Ongoing Content Cadence

Post cadence matches COMMUNITY.md §4 (Monday/Wednesday/Friday rotation).
First-month post drafts are in docs/SOCIAL_REVIVAL_PLAN.md §2 (W1-A through W4-B).

Key rules for every post:

1. Education posture: every post teaches a concept, shows a process, or shares a
   community update. No post implies someone should take a specific market action.
   Apply the COMMUNITY.md §4 filter: "Does this post teach a concept, show a process,
   or share a community story? Or does it imply someone should take a specific market
   action?" — second answer = do not post.

2. No performance language: zero tolerance for win rates, profit figures, earnings
   implications, member outcome showcases (without full FTC disclosure treatment —
   see RISK_REGISTER §17 for the required disclosure block).

3. Footer on every post: "Education, not financial advice. Trading involves risk of
   significant loss. Simulated results are not real trading. No signals, ever."
   Exact text from docs/templates/DISCLOSURE_BLOCK.md §C.

4. Game Devlog posts: do not publish until the vertical-slice build is confirmed
   shipped (ROADMAP Phase 2 exit). Until then, use the Process-Concept variant
   (see docs/SOCIAL_REVIVAL_PLAN.md W3-B and W4-B for examples). All game posts
   state clearly that the game is in development.

5. FTC compliance on any member-outcome content: cohort graduation posts or any
   post referencing a member result require (a) material-connection disclosure
   and (b) typicality disclosure. Use the disclosure block from
   docs/templates/DISCLOSURE_BLOCK.md §A. No showcase without both disclosures.

Quarterly cadence check: every three months, assess whether Facebook is producing
real community engagement (referrals to Discord, genuine reach). If not, reduce cadence
or park the surface rather than posting into a void.
